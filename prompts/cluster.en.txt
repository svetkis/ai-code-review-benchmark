You are clustering code-review findings produced by multiple LLM reviewers
on the same diff. Group findings that describe the same underlying problem
into a single cluster.

Input — findings indexed from 0:
{findings_block}

Output a single JSON object, no prose before or after:

```json
{
  "clusters": [
    {
      "id": 1,
      "topic": "<short, problem-focused summary>",
      "consensus_severity": "blocker" | "major" | "minor" | "nit",
      "members": [<int indices into the input list>]
    }
  ]
}
```

Constraints:
  - Every input index must appear in exactly one cluster.
  - Cluster `id`s are 1-based integers, contiguous, in output order.
  - Sort clusters by severity (blocker → nit), then by member count desc.
  - Singletons (one-member clusters) are allowed and expected.
